Informativa sulla Privacy e Trattamento Dati (Google OAuth)

Data di ultima revisione / Last Updated: 14 luglio 2026

Il Dott. Parrinello Giovanni (di seguito "il Professionista"), in qualità di Titolare del Trattamento, si impegna a proteggere la privacy degli utenti che utilizzano la piattaforma e il software "NUOVA ENERGIA" (accessibile tramite il sito web https://www.nuovaenergianutri.it).

1. Dati raccolti tramite le API di Google (OAuth 2.0) / Data Accessed

Il software sviluppato da Nuova Energia richiede l'autorizzazione dell'utente per accedere ai seguenti servizi Google tramite protocollo sicuro OAuth 2.0:

  • Google Calendar (.../auth/calendar.events): per la sincronizzazione, visualizzazione e gestione degli appuntamenti dello studio, inclusa la lettura dei calendari e la creazione, modifica o aggiornamento dei relativi eventi.
  • Gmail (.../auth/gmail.send): per l'invio automatizzato di promemoria e comunicazioni relative alle visite.
  • Google Forms (.../auth/forms.body, .../auth/forms.responses.readonly): per la creazione di moduli e l'importazione in locale delle risposte e dei dati anamnestici compilati dai pazienti.
Our desktop application accesses the following Google user data based on the permissions granted by the user: raw email content and metadata (via gmail.readonly and gmail.send), Google Calendar event details (via calendar.events), and Google Forms structures and responses (via forms.body and forms.responses.readonly). No aggregated or anonymized tracking data is collected from your Google account.

2. Finalità e conformità all'uso dei dati (Google Limited Use Policy) / Data Use

L'uso delle informazioni ricevute dalle API di Google da parte del software rispetta rigorosamente le Norme sui dati dei servizi API di Google, inclusi i requisiti per l'Uso Limitato (Limited Use Requirements).

I dati vengono utilizzati esclusivamente per fornire le funzionalità operative descritte e non vengono memorizzati su server esterni al dispositivo dell'utente.

Nello specifico:

  • Non condividiamo, vendiamo o trasmettiamo in nessun caso i dati personali a terze parti.
  • Non utilizziamo i dati ottenuti tramite le API di Google per scopi pubblicitari, profilazione commerciale o per il tracciamento del comportamento dell'utente.
  • L'autenticazione avviene sul cloud sicuro di Google; il software non entra mai in possesso della password dell'account.
The application’s use and transfer to any other app of information received from Google APIs will adhere to the Google API Services User Data Policy, including the Limited Use requirements. Google user data is used exclusively to power the local core features of the application (scheduling, local form processing, and email communication) within the local user interface.

2.1 Divieto di Addestramento di Modelli IA/ML / AI/ML Model Training Restrictions

Il software non utilizza, in nessun caso, i dati grezzi o aggregati ottenuti tramite le API di Google Workspace (Gmail, Calendar, Forms) per sviluppare, migliorare o addestrare modelli di Intelligenza Artificiale (AI) o Machine Learning (ML). Nessun dato derivante dai servizi Google viene trasferito a servizi di IA di terze parti.

The application does not use raw, aggregated, or anonymized user data received from Google Workspace APIs to develop, improve, or train any Artificial Intelligence (AI) or Machine Learning (ML) models. Furthermore, no Google Workspace data is transferred to third-party AI services. The use of raw or derived user data received from Workspace APIs adheres strictly to the Google API Services User Data Policy, including the Limited Use requirements.

3. Trasferimento e Condivisione dei Dati / Data Transfer

Il software opera interamente in locale sul computer dell'utente. Non condividiamo, vendiamo, affittiamo o trasmettiamo in nessun caso i dati personali o le informazioni ottenute tramite le API di Google a terze parti, server cloud esterni o piattaforme pubblicitarie.

We do not sell, rent, trade, or transfer any Google user data to third parties. Because this application operates entirely as standalone desktop software, all data fetched from Google APIs remains strictly on the user's local machine. No data is transmitted to our servers or any external third-party services.

4. Protezione e Sicurezza dei Dati / Data Protection

Tutte le interazioni tra l'applicazione desktop e le API di Google avvengono tramite connessioni crittografate che utilizzano protocolli sicuri standard del settore (SSL/TLS) durante il transito. Localmente, i dati sono protetti all'interno dell'infrastruttura di sicurezza del sistema operativo dell'utente. Il software non entra mai in possesso della password dell'account Google.

All interactions between the desktop application and Google APIs are encrypted using industry-standard Secure Socket Layer (SSL/TLS) protocols during transit. Locally, data is restricted within the application's isolated environment and is protected by the security infrastructure of the user’s operating system.

5. Conservazione e Cancellazione dei Dati / Data Retention & Deletion

I dati dell'utente Google non vengono conservati in modo permanente su server esterni. I dati vengono memorizzati nella cache locale solo per il tempo strettamente necessario a eseguire le sessioni attive nel software. L'utente può eliminare definitivamente tutti i dati memorizzati in locale in qualsiasi momento disconnettendo l'account Google dall'app, svuotando la cache o disinstallando il software.

L'importazione dei dati avviene localmente sul dispositivo dell'utente. La cancellazione e la successiva gestione dei moduli sorgente su Google Forms rimangono sotto l'esclusiva responsabilità del professionista utilizzatore, in qualità di Titolare autonomo del Trattamento.

We do not retain your Google data on any external servers. Google user data is cached locally only as long as necessary to execute the active session or workflow within the app. Users can permanently delete all locally stored data at any time by logging out of the Google integration inside the app, clearing the application cache, or uninstalling the software.

6. Diritti dell'utente e revoca del consenso / User Rights

L'utente può revocare in qualsiasi momento l'accesso del software ai propri dati Google direttamente dalle impostazioni di sicurezza del proprio Account Google (sezione "App di terze parti con accesso al conto"), oppure interrompendo l'uso delle funzioni integrate.
